# Вы работаете в известной компании, занимающейся полётами в космос - SpaceY. Сам директор компании, знаменитый
# изобретатель Дилан Маск, поручил вам спроектировать новую систему. Есть завод по массовому производству и сборке
# ракет для полета в космос и выполнения разных миссий. В ракетах присутствуют разные двигатели, разные ступени,
# разные топливные отсеки, разная аппаратура для разных миссий. Директор компании хочет упростить этот процесс таким
# образом чтобы Ваша система при нажатии пары кнопок в приложении прогоняла весь процесс - от сборки ракеты до её поставки
# на космодром.
#
# На первом этапе требуется только прототип системы.
#
# Задача - спроектировать систему классами. При выборе определённого шаблона проектирования объяснять выбор.

Для решения задачи совмещены два шаблона - Фабричный метод для определения процесса сборки ракеты, поставки её на
космодром и прочих методов, общих для ракет (например процедуры старта).
Фабричный метод был выбран в связи с тем, что у всех ракет, выпускаемых с завода, должна быть общая логика действий -
сборка, тестирование, доставка до стартового стола, предстартовые процедуры и тд. Фабричный метод позволит реализовать 
эти требования максимально предсказуемо. 
Кроме того использована Абстрактная фабрика для генерации деталей ракеты. Абстрактная фабрика позволит сделать генерацию
компонентов ракеты предсказуемой, с общими интерфейсами для выделяемых семейств. 
"Сборка" ракеты композицией.
